# Brain Unfog — UI/UX 개선 제안

> 평가 시점: 2026-04-29
> 평가 관점: 시니어 UI 디자이너 / 산업 디자이너
> 대상: 캘린더 뷰, 간트(프로젝트) 뷰, 편집 인스펙터, 추가 팝오버, 캘린더/프로젝트 메뉴

---

## 0. 총평

기능 설계는 야심차고 — 할일·일정·프로젝트·간트를 한 도구 안에 묶어내려는 시도가 분명합니다. 그러나 현재 시각 시스템은 **"무엇이 더 중요한가"를 사용자에게 말해주지 못하고** 있습니다. 컬러·아이콘·텍스트 무게가 모두 비슷한 수준에서 경쟁하기 때문에, 화면이 정보 밀도에 비해 *읽히지* 않습니다.

핵심 진단을 한 줄로 요약하면:

> **"디자인의 75%를 차분하게 만들고, 25%만 강조하라"** — 현재는 100%가 동등하게 강조되어 있습니다.

아래 제안들은 이 원칙을 구현하기 위한 실행 항목입니다. 우선순위는 **P0 (즉시) → P1 (단기) → P2 (중기) → P3 (탐색)** 순입니다.

## 1. 글로벌 디자인 시스템 (P0)

### 1.1 컬러 토큰 분리

현재 시스템 색과 시멘틱(프로젝트/캘린더) 색이 섞여 있어, 녹색이 "추가 버튼", "믹싱 July 캘린더", "체크 완료"에 동시에 쓰입니다. 의미가 분산됩니다.

**제안:**
- **System Colors**: 5단계만 사용 — `accent` (브랜드 1색, 추천 보라/인디고), `neutral-*`, `success`, `warning`, `danger`. 시스템 액션(버튼·링크)은 오직 `accent`만.
- **Semantic Colors**: 프로젝트·캘린더는 별도 팔레트. 채도를 시스템 색보다 한 단계 낮춰 시각적 우선순위를 시스템 < 콘텐츠 < 사용자 액션 순으로.
- 툴바의 ➕(녹색)와 ↪(주황)은 너무 saturated. → 두 버튼 모두 `accent` 단색으로 통일하고, 의미 차이는 모양으로.

### 1.2 컬러 사용량 70% 감축

캘린더의 ALL-DAY 영역을 보면 8~10개 항목이 각자 다른 컬러로 채워져 있어서 **눈이 길을 잃습니다**. 컬러는 "분류 식별"이 아니라 "주의 환기"의 도구로 재정의해야 합니다.

**제안:**
- 이벤트 블록 본문은 중성(`neutral-50` 배경 + `neutral-900` 텍스트)으로.
- 프로젝트/캘린더 색은 **좌측 4px 인디케이터 바** 추가.
- 채워진 컬러 배경은 *오늘*, *현재 진행 중*, *충돌* 같은 강조 상황에만.

### 1.3 타이포그래피 위계 3단 강제

```
Title    : SF Pro 14pt  / 600 / neutral-900
Body     : SF Pro 13pt  / 400 / neutral-800
Meta     : SF Pro 11pt  / 400 / neutral-500
```

현재는 본문과 메타("사적인", "구글사적인", "대한민국 공휴일")의 시각 차가 거의 없어 모두 본문처럼 읽힙니다. 메타는 한 단계 더 약하게(`neutral-500` 또는 `neutral-400`).

### 1.4 아이콘 시스템 통일

현재 혼재된 스타일:
- 체크박스: 외곽선만(○) vs 채워짐(●)
- 반복 표시: 양방향 화살표(⇄)
- 캘린더 카테고리 도트: 채움 원
- 프로젝트 행 좌측: ⚡(번개) vs ⏰(시계) vs 빨간 점 vs 88격자

**제안:** SF Symbols 기반으로 두 가지 굵기(`regular`/`medium`)만 사용. 동일 의미는 동일 모양 — 예: 모든 "할일"은 `circle`, 완료는 `checkmark.circle.fill`로 일관.

## 2. 캘린더 (Day/Week) 뷰 (P0–P1)

### 2.1 ALL-DAY 영역 정리 (P0)

현재 한 줄에 체크박스 할일과 컬러 도트 이벤트가 섞여 9개 이상 쌓여 있어 시각적 노이즈가 가장 심한 영역입니다.

**제안:**
1. **할일과 이벤트를 하위 행으로 분리** — 상단에 이벤트(컬러 도트 표시), 그 아래 할일(체크박스 표시). 사이에 1px 디바이더.
2. **공휴일은 별도 띠**로 격하 — "노동절 대한민국 공휴일"이 두 번 반복 표시되는 것은 데이터 중복. 한 번만 표시하고 reduced contrast로.
3. **펼침/접힘 토글** — 4개 초과 시 "+3 더보기" 패턴. 전체 펼치기는 사용자 선택.
4. ALL-DAY 라벨을 90° 회전 또는 좌측 상단 한 곳에만 배치(현재처럼 세로 가운데 정렬은 OK이나 좀 더 작게).

### 2.2 시간 영역 가독성 (P1)

- **현재 시간 빨간 라인**에 *시간 라벨 칩*(예: `11:24`)을 좌측에 부착해 정보성 강화.
- 30분 라인을 더 약하게(현재는 정시 라인과 거의 같은 톤).
- **빈 시간 슬롯에 호버 시** 미세한 highlight + "➕ 새 할일/일정" 가이드. 현재는 클릭 후 팝오버가 떠야 알 수 있음.
- 점심·저녁 시간대(12·6PM) 미세한 백그라운드 톤 차이로 시간 감각 보조.

### 2.3 이벤트 블록 디테일 (P1)

- 시간 텍스트 `09:30–12:22` 의 dash는 en-dash(–)인데 자간이 헐거움 → `–` 좌우 하프스페이스 또는 fixed-width.
- 캘린더 이름("구글사적인")이 본문 사이즈와 같음 → 메타 사이즈로 격하.
- 반복 아이콘(⇄)은 우측 상단 코너에 미세하게(`neutral-400`). 현재는 본문 옆에 있어 어수선함.
- **블록 모서리** 6px → 4px로. 현재는 너무 둥글어 카드 같지 않고 알약 같음.

### 2.4 사이드바·캘린더 분리 창 통합 (P0)

좌측 "괴래 새 시나리오" 플로팅 윈도우는 **macOS 네이티브 멘탈 모델과 충돌**합니다. 사용자는 두 창의 관계를 학습해야 하고, 어두운 데스크탑 위에서 두 창이 더 도드라져 보입니다.

**제안 A (권장):** 메인 윈도우 좌측에 **접을 수 있는 사이드바**로 통합. macOS 표준 NavigationSplitView 패턴.

**제안 B:** 별도 창을 유지하되 — (a) 보조 창은 액센트 컬러 빼고 무채색 톤, (b) 메인 창과 도킹·스냅 가능, (c) 사용자 선호로 통합 모드 토글.

## 3. 인스펙터(편집 패널) (P0)

### 3.1 점유율 문제

현재 우측 편집 패널이 창 폭의 약 35%를 차지하나, 표시되는 정보는 제목 1줄 + 빈 내용 영역 + 빈 첨부 + 날짜 정도. **시각적 무게에 비해 정보 밀도가 너무 낮습니다**.

**제안:**
- 너비 320–360px로 고정 (현재 ~600px 추정). 콘텐츠 정렬도 좌측 정렬로 좁힘.
- 또는 **Floating Inspector** (Sketch/Figma처럼 띄움 가능) — 큰 창에서는 도킹, 작은 창에서는 자동 floating.
- 첨부파일 드롭존을 **48px 작은 영역에서 시작**, 드래그 진입 시에만 확장.

### 3.2 폼 레이아웃

- "제목" 입력은 **타이틀처럼** 보이도록 18pt + 라벨 제거. 현재는 일반 인풋.
- "내용" 빈 영역이 너무 큼 → 자동 높이(min 80px, 입력에 따라 확장).
- "날짜 / 시간 설정" 체크박스 + 인라인 캘린더는 비효율 — **텍스트 입력 + 팝오버 캘린더** 패턴으로 (Apple Calendar / Things 3 방식).
- "반복 일정 적용 범위 [이 일정만 / 이후 반복 일정]" 세그먼트는 **충돌 시 모달**로 분리하는 것이 사고 사고를 줄임.

### 3.3 닫기/저장 흐름

- 우측 상단 X 버튼 외에 **Esc**, **⌘Return**(저장 후 닫기) 등 단축키 명시.
- 변경 사항 자동 저장이라면 인디케이터(예: "저장됨" 짧은 토스트), 수동이라면 **Save 버튼 명시** — 현재는 어느 방식인지 불분명.

## 4. 툴바 (P1)

### 4.1 의미 명확화

```
[📊][📅]   [➕][↪]   [오늘]   [-][+]   [🔍 전체 검색]
```

- 좌측 `📊 / 📅`는 뷰 전환(간트/캘린더). **세그먼티드 컨트롤** 형태로 결합 + 라벨("간트", "캘린더").
- `➕`(추가)와 `↪`(가져오기?)의 의미가 불명확 → 호버 툴팁 + 아이콘 재고려.
- `-`/`+` 줌 컨트롤은 위치/맥락이 모호 → 우측 하단 zoom control로 옮기거나 키보드 단축키만.
- **검색 인풋**은 너무 넓음 — 작은 검색 아이콘 → 클릭 시 확장 패턴.

### 4.2 좌상단 캘린더 필터 진입점

스크린샷 11.43.50에서 본 캘린더 필터 메뉴는:
- 14개 캘린더 + 활성/표시만/숨김 3-state 푸터 — **정보가 많은데 진입 아이콘은 작은 캘린더 도트만**.
- 호버 시 어떤 컨트롤인지 명확하지 않음 → 라벨 또는 chevron 추가.
- 필터 상태(몇 개 활성?)를 진입점에 배지로 표시 — 예: `📅 9/14`.
- 정렬: 현재 "가족·고집·구글사적인…" 가나다 순. **자주 쓰는 항목 상단 고정** 옵션.

### 4.3 윈도우 크롬

- 시그널 라이트(빨/노/녹)와 사이드바 토글이 **타이틀바 라인보다 아래**에 떠 있음 → macOS 표준 위치로 정렬.
- "Brain Unfog" 타이틀 폰트가 시스템 default와 어색하게 큼 → 13pt regular.

## 5. 추가 팝오버 (할일 추가) (P0)

### 5.1 두 가지 변형의 일관성

현재 두 가지 팝오버가 존재합니다:
1. **컴팩트** (툴바 ➕ 클릭): 프로젝트 드롭다운 `2026 ⌃` + 인풋 + [취소][추가]
2. **풀** (캘린더 슬롯 클릭): 인풋 + 프로젝트 드롭다운 `Live -------` + [취소][추가]

순서가 뒤바뀌어 있습니다. **인풋 → 프로젝트** 또는 **프로젝트 → 인풋** 중 하나로 통일.

### 5.2 "Live -------" placeholder 문제

대시(`-------`)가 placeholder로 보이는데 **깨진 UI 또는 로딩 실패처럼 읽힙니다**. 실제로는 "Live (영구)" 같은 의미인 듯한데:
- 명시적 라벨로 변경 — 예: "Live · 기한 없음" 또는 "프로젝트 선택"
- 빈 상태일 때 회색 italic placeholder

### 5.3 슬롯 클릭 → 팝오버 앵커링

스크린샷 11.32.46에서 빈 시간 슬롯 클릭으로 뜬 팝오버가 **앵커 화살표 없이** 그 자리에 박혀 있습니다. 어떤 시간에 추가되는지 시각적 연결이 약함.

**제안:**
- 팝오버에 ▲ 화살표 — 클릭한 슬롯을 가리키도록.
- 팝오버 내부에 시간 미리보기 — 예: "수 4/29 오전 11:00–12:00" 작은 텍스트.
- **시간 직접 편집 가능** — 인풋 옆에 시간 chip.

### 5.4 프로젝트 드롭다운 (스크린샷 11.43.39)

20개 이상의 프로젝트가 알파벳/가나다 순 단일 리스트로 노출 — 길어짐. 또한:
- "2026"이 맨 위, 그 아래는 가나다 — 정렬 규칙이 혼합됨. 명시적인 "최근 사용" 섹션 분리 권장.
- 프로젝트별 **컬러 도트** 미표시 → 캘린더의 컬러 시스템과 연결 끊김. 좌측에 작은 도트 추가.
- 검색 가능한 콤보박스로 — 20+ 항목에선 필수.

## 6. 간트 / 프로젝트 타임라인 뷰 (P1)

이 뷰가 현재 가장 잘 디자인되어 있습니다. 다만 몇 가지:

### 6.1 빈 행 처리

"구매", "경제적 자유", "차량 수리", "영화 보기" 등 **0개 항목인 프로젝트가 4–6행 연속**으로 비어 있습니다. 화면이 죽어 보이는 인상.

**제안:**
- 빈 프로젝트 자동 접기 토글 — "보기 옵션 → 빈 프로젝트 숨김"
- 또는 빈 행 높이를 절반으로 (현재 28px → 16px)

### 6.2 셀 안 숫자 배지

`①②③` 같은 카운트 배지에 호버해야 항목명이 뜨는 듯한데, 정보 접근 비용이 높음.

**제안:**
- 작은 도트 미리보기 옵션 — 호버 없이 한 줄 더보기 표시
- 클릭 시 팝오버(스크린샷 11.31.42처럼)는 좋음, 다만 팝오버 위치를 셀에 정확히 앵커링

### 6.3 프로젝트 진행률

좌측 `8/9`, `1/2`, `3/4` 같은 분수 표기가 직관적이지 않음. **미니 프로그레스 바**(8px 높이) + 분수 텍스트 병기 권장.

### 6.4 우클릭 컨텍스트 메뉴

스크린샷 11.44.19에서 본 [이름 변경 / 색상 / 숨김 / 삭제] 메뉴는 깔끔. 다만:
- "삭제"가 마지막에 있는 건 OK, 그러나 색상으로 구분 필요(빨간 텍스트) — 실수 방지.
- 디바이더 위치: "삭제" 위에 한 번 더 디바이더(파괴적 액션 격리).

### 6.5 가로 스크롤 인디케이터

현재 4월에서 5월 11일까지 약 6주가 보이는데, **양 끝에 페이드 마스크**(8px gradient)로 더 있다는 시각적 hint를 주면 좋음.

### 6.6 헤더 날짜 컬럼

`월 화 수 목 금 토 일 / 27 28 29 30 1 2 3` — 주말은 빨/주황으로 강조되어 있는데, **공휴일 강조와 시각 충돌**. 주말은 회색조, 공휴일만 빨간색이 macOS Calendar와 일관.

## 7. 인라인 폼 (P2)

스크린샷 8.39.54의 인라인 추가 폼:

```
○ 이거는
⊕ 새 할일 [____________]
```

- `⊕` 아이콘이 너무 진하고 큼 — 보통 inline add는 `+` 텍스트 또는 작은 아이콘.
- "새 할일" placeholder가 italic이 아니어서 입력값과 구분 안 됨 → italic 또는 더 옅게.
- Enter로 추가 후 폼이 유지되는지 사라지는지 명시적 — 보통 **유지 + 새 빈 줄**(연속 입력 패턴)이 좋음.

## 8. 마이크로 인터랙션 (P2)

현재 정적 스크린샷만으로는 단정 어렵지만, 다음 모멘트들이 핵심 후보:

| 상황 | 제안 모션 |
|---|---|
| 할일 체크 | 200ms ease-out 컬러 채움 + 80ms strikethrough 슬라이드 |
| 이벤트 블록 드래그 | 3° tilt + 6% scale + 그림자 강화 |
| 인스펙터 열림 | 250ms slide-in (우→좌) + 콘텐츠 fade 80ms 지연 |
| 추가 팝오버 | 150ms scale 0.96→1.0 + opacity 0→1 |
| 현재 시간 라인 | 1분마다 부드럽게 `top` 이동 (jump 없음) |
| 간트 호버 행 | `bg-neutral-50` 100ms |

> 모든 애니메이션은 **`prefers-reduced-motion` 존중**.

## 9. 접근성 (P1)

### 9.1 컬러 대비

- "사적인", "대한민국 공휴일" 같은 메타 텍스트가 매우 옅음 → WCAG AA(4.5:1) 미달 가능. 검증 필요.
- 일부 컬러 이벤트 블록의 본문 텍스트 — 옅은 파스텔 배경 위 검정 텍스트지만, 상태별 호버에서 깨질 수 있음.

### 9.2 키보드 내비게이션

- 캘린더 그리드: 방향키로 시간 슬롯 포커스 이동
- 인스펙터: Tab 순서 명시 (제목 → 내용 → 날짜 → 시간 → 첨부 → 닫기)
- 팝오버: Esc 닫기 + Return 추가

### 9.3 VoiceOver 라벨

- 컬러 도트만으로 분류를 표현하는 곳은 모두 텍스트 라벨 필요. 예: "사적인 캘린더, 청록색"
- 진행률 `8/9`는 "9개 중 8개 완료"로 읽혀야 함

## 10. 다크 모드 검토 (P2)

현재 스크린샷은 라이트 모드만 보임. 다크 모드 시:
- 채도 낮춘 컬러 토큰 별도 정의 (라이트 컬러를 그대로 어두운 배경에 올리면 진동)
- 캘린더 그리드 라인은 `neutral-700` 정도
- 데스크탑이 어두울 때 윈도우 그림자가 필요(현재 라이트에선 그림자가 약한지 거의 안 보임)

## 11. 즉시 실행 가능한 Top 7 (요약)

화면 단위 변화가 가장 큰 일곱 가지만 추리면:

1. **ALL-DAY 영역 분리** — 이벤트/할일 하위 행으로 나누고 공휴일 격하 (P0)
2. **이벤트 블록 컬러 변경** — 컬러는 좌측 4px 바, 본문은 중성 (P0)
3. **인스펙터 폭 320–360px로 축소** + 폼 컴팩트화 (P0)
4. **두 창 통합 또는 보조 창 무채색화** (P0)
5. **"Live -------" placeholder 명시 라벨로 교체** (P0)
6. **툴바 ➕/↪ 아이콘 액센트 통일** + 의미 라벨 (P1)
7. **타이포그래피 3단 위계 강제** — Title/Body/Meta (P0)

이 일곱 가지만 적용해도 화면의 "읽힘"이 체감 가능 수준으로 좋아질 것입니다.
