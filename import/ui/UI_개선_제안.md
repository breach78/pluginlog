## 1. 절대 건드리지 않을 것 (Sacred Cows)

다음 항목들은 **의도된 디자인**이며, 본 제안에서 보존·강화 대상입니다.

### 1.1 할일 동그라미 아이콘 시스템
- 동그라미 안에 들어가는 변형들(반복 할일, 지난 할일 표식 등)은 본인이 직접 커스텀했고 마음에 들어함.
- 캘린더 뷰의 작은 표식 아이콘들(반복·지난)도 동일.
- → **유지**, 단 굵기/사이즈 토큰만 시스템에 편입(2.3 참조).

### 1.2 프로젝트 우선순위 시각화 (아이젠하워 매트릭스)
- 좌측 시나리오 창 / 간트 행에서 **위에 있을수록 큰 아이콘 = 중요**, 아래로 갈수록 작아짐.
- 같은 모양 아이콘으로 분류군을 묶고 우선순위를 위치+크기로 표현 — 본인이 가장 자랑스러워하는 디자인.
- → **유지·강화**. 4-1단에서 더 또렷하게 표현하는 방법 제안.

### 1.3 ALL-DAY 영역의 "오늘 인박스" 역할
- 할일·이벤트·공휴일이 섞인 것은 의도. 이 영역은 **오늘 할 일이 모이는 인박스**이고, 여기서 시간 슬롯으로 끌어내려 배치하는 것이 핵심 워크플로우.
- → **유지**. v1에서 제안했던 "할일/이벤트 분리"는 **취소**.

### 1.4 우측 편집 패널의 큰 폭
- 의도적: 오른쪽 패널에서 할일을 크게 보며 직접 작업.
- → **유지**. v1에서 제안했던 "320–360px로 축소"는 **취소**.

### 1.5 호버 미리보기
- 마우스 호버 시 할일 미리 보여주는 인터랙션은 잘 만들어진 부분.
- → **유지·확장**. 다른 영역(간트 셀, 프로젝트 행)에도 동일 패턴 적용 검토.

### 1.6 다크모드 / 두 창 분리
- 다크모드: **고려 안 함**.
- 두 창 분리(좌측 작은 창 + 메인): 의도(여러 프로젝트 동시 보기). **유지**. 단 본인이 "오른쪽 패널 아래에 합칠까" 고민 중인 점은 6장에서 언급.

## 2. 디자인 언어 정렬 (P0)

목표: **Things 3의 여백·차분한 위계** + **Apple Calendar의 컬러·동작**의 결을 일관되게.

### 2.1 컬러: Apple Calendar 결을 따르기

Apple Calendar의 이벤트 블록은:
- 캘린더 컬러를 **연한 채도(약 15~20% opacity 배경 + 100% 좌측 막대 또는 도트)** 로 표현
- 이벤트 텍스트는 캘린더 컬러보다 채도 높은 진한 톤
- 흰 바탕은 충분히 살림 (배경이 컬러로 다 덮이지 않음)

**현재 앱과 비교:**
- 일부 블록(스킨답서스 물, 의사면담 등)은 채도가 적정
- 일부(시라트 상상마다, 수영 점심)는 배경 채도가 너무 높아 텍스트 가독성 저하 직전
- 컬러 일관성이 약함 — 같은 캘린더(예: 사적인)인데 블록마다 톤이 다른 인상

**제안:**
- 모든 이벤트 블록을 동일 공식으로 — `bg: tint @ 14%`, `text: tint shifted -20% lightness`, `좌측 3px bar: tint @ 100%`
- ALL-DAY 영역의 컬러 도트도 동일 톤 체계
- 본인이 만족하는 Apple Calendar의 정확한 톤을 OS Sequoia 기준으로 샘플링해 토큰화

### 2.2 타이포: Things 3 결을 따르기

Things 3는 **글씨 크기와 굵기 차이를 크게 두지 않고**, 색·여백으로 위계를 만듭니다.

**제안 (3단만 사용):**
```
Title    SF Pro 13.5pt / 510(medium) / #1A1A1A
Body     SF Pro 13pt   / 400(regular) / #2C2C2E
Meta     SF Pro 11pt   / 400(regular) / #8E8E93
```

- 현재 메타 텍스트("구글사적인", "사적인", "대한민국 공휴일")가 본문과 거의 같은 톤 → Meta 토큰으로 격하.
- 시간 텍스트 `09:30–12:22`는 **Meta 톤 + tabular numbers**로.
- "ALL-DAY" 라벨은 9pt / uppercase / `#C7C7CC`로 더 작고 약하게.

### 2.3 아이콘 굵기 토큰 통일

본인이 마음에 들어하는 할일 동그라미·반복·지난 아이콘은 유지하되, **외곽선 굵기를 1.5pt 단일 값**으로 정렬. 현재는 일부 1pt, 일부 2pt가 섞여 있어 같은 크기에서도 시각 무게가 다릅니다.

### 2.4 코너 라운딩

Things 3의 카드는 6–8pt 라운딩(부드럽지만 알약은 아님). 현재 이벤트 블록은 일부 12pt에 가까워 알약처럼 보임 → **6pt로 통일**.

## 3. 캘린더 (Day/Week) 뷰

ALL-DAY 영역의 역할(인박스)은 유지하되, **시각적 완성도**만 정돈합니다.

### 3.1 ALL-DAY 영역 — 정돈하되 분리는 안 함

- 모든 항목 라인 높이 22px로 통일 (현재는 18~24 혼재되어 보임)
- 좌측 인디케이터(체크박스 / 컬러 도트)의 **세로 정렬 기준선** 일치
- 같은 줄에 본문과 메타가 있을 때 본문↔메타 사이 6pt 고정 간격
- 공휴일 두 줄 중복("노동절 대한민국 공휴일" / "노동절 대한민국의 휴일")은 데이터 중복 — **하나만 표시** (이건 시각이 아니라 데이터 정리이지만 화면 청결도에 큰 영향)

### 3.2 시간 그리드

- 30분 라인 색을 정시 라인의 **40%** 톤으로 (현재는 거의 같음)
- 현재 시간 빨간 라인 좌측에 **시간 칩**(예: `11:24`) 추가 — Things가 아니라 Fantastical/Cron의 좋은 디테일
- 점심·저녁 시간대(11–13, 17–19)는 미세한 배경 워시 (`#F8F8F9` 정도) — 시간 감각을 시각화

### 3.3 이벤트 블록 디테일

- 좌측 컬러 막대 **3pt 두께 / 블록 높이 100%** (현재는 일부 블록만, 또는 없음)
- 시간 텍스트의 dash는 en-dash(`–`) 고정폭 변형으로
- 반복 아이콘(⇄)은 우상단 corner에 `#8E8E93`로 옅게
- 호버 시 그림자 1단계 상승 (`0 1px 2px rgba(0,0,0,0.04)` → `0 4px 12px rgba(0,0,0,0.08)`) — Things 3의 호버 결

### 3.4 캘린더 헤더

- 요일 한글 약어("수", "목", "금", "토", "일")의 **자간을 좁히고** 11pt
- 날짜 숫자 17pt / 510 / 오늘 날짜는 컬러 채움 원 안에 흰 숫자 (Apple Calendar 동일)
- 주말 색은 회색조 — 공휴일과 충돌하지 않게. 공휴일만 빨강으로 강조

## 4. 프로젝트 / 간트 뷰

### 4.1 우선순위 시각화 강화 (본인의 자랑 시스템)

현재 좌측에서 ⚡(중요?) → ⏰(시간성?) → 빨간 점(긴급?) → 88격자(일반)로 분류하고 있는 듯합니다. 이게 **아이젠하워 매트릭스 의도**라면 다음으로 더 또렷해집니다:

- 분류군 사이에 **여백 8pt**로 그룹 시각화 (현재는 행 사이 간격이 균일해서 분류가 묻힘)
- 각 분류군 시작에 **소제목**(11pt / `#8E8E93` / uppercase)
  ```
  IMPORTANT & URGENT
  ⚡ 2026                8/9
  ⚡ 괴래 새 시나리오       2

  IMPORTANT
  ⏰ 성균관대학교          3/4

  URGENT
  ● 앱 수정              4/4

  GENERAL
  ▦ Live                24
  …
  ```
- 또는 본인 의도에 맞는 한국어 라벨로 ("중심", "현재", "긴급", "일반" 등)
- 분류 아이콘 자체는 본인이 만든 그대로 유지

### 4.2 진행률을 시각화

`8/9`, `3/4`, `1/2` 분수 표기 → **분수 텍스트 + 그 아래 2px 미니 프로그레스 바**(프로젝트 컬러). Things의 프로젝트 원형 진행도와 다른, 가로 막대 형태가 간트와 잘 어울림.

### 4.3 빈 프로젝트 행

빈 프로젝트("구매", "경제적 자유", "차량 수리" 등)가 연속 4–6행 — 화면이 비어 보임.
- 빈 행 높이를 28px → 22px로
- 또는 **"빈 프로젝트 접기" 토글** (보기 옵션)

### 4.4 호버 미리보기 광역화

본인이 좋아한다는 호버 시 미리보기를 **간트 셀의 숫자 배지**(`①②③`)에도 동일 적용.
- 호버 → 작은 카드(220px 폭)에 항목명 리스트 + 체크박스
- 모션: 100ms ease-out, 4pt 위로 떠오름

### 4.5 헤더 날짜

`월 화 수 목 금 토 일 / 27 28 29 30 1 2 3` 헤더에서:
- 주말은 회색조, 공휴일만 빨강 (3.4와 동일)
- 오늘 컬럼 전체에 미세한 컬러 워시 (현재 세로 라인은 좋음, 추가로 배경)
- 월 경계(4월 30 → 5월 1)에 더 진한 vertical divider — 월이 바뀌는 시각적 신호

## 5. 인스펙터 (편집 패널)

폭은 **유지**합니다(본인 의도). 내부 폼만 정돈합니다.

### 5.1 폼 위계

- "제목" 라벨 제거, 인풋 자체를 **제목 처럼** 18pt / 510으로 — Things 3의 task 디테일 결
- "내용" 라벨도 제거. placeholder만(`내용을 입력하세요…`)
- "첨부파일" 드롭존: 32px 높이로 시작, 드래그 진입 시 확장 — 현재 큰 빈 영역의 시각적 무게 감소
- "날짜 / 시간 설정" 체크박스는 그대로, 단 인라인 캘린더가 항상 펼쳐져 있는 대신 **날짜 텍스트 → 클릭 시 팝오버**

### 5.2 반복 일정 / 적용 범위 세그먼트

"적용 범위 [이 일정만 / 이후 반복 일정]" 세그먼트는 좋음. 단:
- 세그먼트 색을 시스템 액센트 단색으로 (현재는 컬러 다소 강함)
- 여백을 위/아래 12pt 확보

### 5.3 닫기 / 자동 저장 시그널

- 자동 저장이라면 우상단에 **점멸하지 않는 작은 체크마크**(`#34C759` / 9pt) — "저장됨"
- Esc 키로 닫기, ⌘W는 윈도우 닫기 (시스템 표준)

### 5.4 빈 상태의 시각적 무게

내용·첨부가 비었을 때 빈 영역이 너무 큼. **컴팩트 모드** 자동 — 비어 있으면 줄어듦, 클릭하면 확장.

## 6. 좌측 보조 창 (괴래 새 시나리오)

본인이 "오른쪽 할일 페인 아래에 합쳐버릴까" 고민 중이라 한 부분. **결정 전 양쪽 모드 모두 살리는 방안**:

### 6.1 분리 유지 시 (현재)
- 보조 창 크롬을 **메인 창과 시각 차별화** — 보조 창은 윈도우 컨트롤(빨/노/녹) 아래 살짝 어두운 톤(`#FAFAFA`)
- 그림자를 연하게(`0 8px 32px rgba(0,0,0,0.08)`) — 띄워져 있다는 신호
- 보조 창 안에서는 **항목명 + 체크박스 + 날짜만** — 더 미니멀하게

### 6.2 통합 모드 옵션 추가
- 보기 옵션: "프로젝트 패널 위치: 분리 창 / 우측 패널 하단 / 좌측 사이드바"
- 본인의 워크플로우에 따라 토글 가능 — **합칠지 말지 결정 전 양쪽 다 살아 있는 게 안전**

## 7. 추가 팝오버

본인 사용 빈도 1순위가 "할일 빠르게 추가"이므로 **이 팝오버는 시각적으로도 가장 정제되어야** 합니다.

### 7.1 두 변형의 일관성

현재 두 가지 형태:
- **컴팩트** (툴바 ➕): 프로젝트 드롭다운 → 인풋 → 버튼
- **풀** (캘린더 슬롯 클릭): 인풋 → 프로젝트 드롭다운 → 버튼

순서를 통일. **인풋을 항상 최상단** (즉시 타이핑 가능 — Things 3 quick entry 원칙).

### 7.2 "Live -------" 정리

본인 답변: 시각적 구분용일 뿐 의미 없음.
- 대시 제거
- 대신 드롭다운 안에서 **분류군 사이 1px divider + 여백 6pt**로 시각적 구분 (4.1과 같은 결)
- 또는 분류군 헤더 라벨

### 7.3 컴팩트 팝오버의 ▲ 앵커

툴바 ➕에서 떴을 때 ▲ 화살표가 ➕ 버튼을 가리키도록. 캘린더 슬롯 클릭 시는 그 슬롯을 가리키도록. **클릭한 곳과 결과가 시각적으로 연결**되어야 합니다.

### 7.4 프로젝트 드롭다운 (스크롤 길이 문제)

20+ 항목일 때:
- 좌측 작은 컬러 도트(2.1의 시멘틱 컬러) 추가
- 상단에 검색 인풋 (8+ 항목부터 자동 표시)
- "최근 사용 3개"를 상단에 고정 섹션

## 8. 툴바 / 윈도우 크롬

### 8.1 ➕ / ↪ 버튼

현재 채도 높은 녹색·주황 → Things 3 / Apple Calendar 결과 충돌. **둘 다 시스템 액센트 단색**(예: 보라/인디고)으로 통일하고 의미는 모양으로 구분. 색 강도는 한 단계 낮게.

### 8.2 뷰 전환

좌측 `📊 / 📅` (간트 / 캘린더) → **세그먼티드 컨트롤** + 작은 라벨 (Things의 inbox/today/upcoming 토글 결).

### 8.3 "오늘" 버튼 + `-/+` 줌

- 둘 다 같은 캡슐 형태로 통일
- 줌 컨트롤은 **우측 하단 zoom**으로 옮기는 것이 시각적으로 깔끔 (현재는 검색창 옆에서 시각적 분리감 부족)

### 8.4 검색 인풋

너무 넓음 → 작은 검색 아이콘 → 클릭 시 가로 확장. Spotlight·Things 3 결.

### 8.5 macOS 표준 크롬

- 시그널 라이트(빨/노/녹)는 **타이틀바 라인에 정확히 정렬**
- 사이드바 토글 아이콘 위치도 시그널 라이트와 같은 baseline
- "Brain Unfog" 타이틀 13pt regular (현재 다소 큼)
